## Core Concept: Wrapper vs. Harness

### Step 1: Understand the Architecture Difference
- **OpenClaw (Wrapper):** Indie developer layer around a model — basic functionality, quick setup, but performance plateaus as complexity increases
- **Claude Code (Harness):** Anthropic's billion-dollar system giving the model file reading, web search, sub-agents, deep research — compounding benefits with no ceiling found yet
- Analogy: OpenClaw puts a Ferrari engine in a Honda chassis; Claude Code IS the Ferrari

---

## Building Your Context OS

### Step 2: Set Up Structured Memory
- Create a structured documentation system that Claude Code can query
- Organize by topic: business data, processes, contacts, decisions
- More comprehensive than OpenClaw's opaque memory system

### Step 3: Centralize Business Data
- Set up a database with all your business data
- Pull from all data sources into one queryable place
- Claude Code can search across this entire knowledge base

### Step 4: Use Markdown Files for Knowledge Transfer
- Package learnings as markdown files
- Share via AirDrop — other person's agent can install the functionality
- Creates a portable knowledge system

## Model Selection for Cost Control

### Step 5: Route Tasks by Complexity
| Task Type | Model | Cost |
|-----------|-------|------|
| Cheap orchestration, simple queries | Haiku | Lowest |
| Medium complexity, routine work | Sonnet | Medium |
| Complex analysis, critical decisions | Opus | Highest |

- Use Haiku for orchestration tasks to keep costs down
- Reserve Opus only for complex analysis where quality matters

## Common Pitfalls
- **Choosing OpenClaw for long-term business automation:** Performance plateaus as complexity scales
- **Using Opus for every task:** Control costs by routing cheap tasks to Haiku
- **Not building a Context OS:** Without structured memory + database, you lose Claude Code's biggest advantage
- **Underestimating setup barrier:** Claude Code requires more initial setup than OpenClaw, but the investment compounds

## Summary
Claude Code's harness (file access, web search, sub-agents) is categorically more capable than OpenClaw's indie-built equivalent. OpenClaw plateaus as complexity increases; Claude Code compounds. Build a "Context OS" (structured memory + documentation + centralized database) for maximum leverage. Use Haiku for cheap orchestration, Opus only for complex analysis. The setup barrier is real but surmountable with templates and community support.
