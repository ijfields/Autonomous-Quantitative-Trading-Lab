### The 5-Question Agent Creation Framework
1. What is the specialty area this agent should focus on?
2. What is the specific objective this agent should achieve?
3. What KPIs will this agent be rated on?
4. What should this agent's personality be like?
5. What tools or data sources does this agent need access to?

Then: "Based on my answers, create a detailed agent persona with a name, emoji, role definition, daily responsibilities, success metrics, and communication style."

### Architecture Rules
- One coordinator agent handles all communication with the human
- Specialists communicate through coordinator, not directly
- Each agent owns its own domain — no overlap
- Define clear handoff processes between agents
- Start with one specialist, master it, then scale

### Common Mistakes
1. Making specialists too narrow (e.g., "only write tweets about crypto" vs. "X content strategy with focus on engagement")
2. No clear handoff process between agents
3. Letting specialists overlap (causes duplicate responses)
