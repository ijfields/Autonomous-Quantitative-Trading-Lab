## What This Guide Covers

How to use Orchids AI (orchids.app) to build a full-stack application from scratch using its desktop client, iterative AI prompting, Claude Code terminal integration, and desktop-to-web sync.

---

## Step 1: Install the Orchids Desktop Client

1. Navigate to orchids.app in your browser.
2. Log in to your account (or create one).
3. Click the download button in the upper right corner.
4. Choose your operating system (Windows, Mac, or other supported platform).
5. Download and run the installer.
6. Open the desktop app and log in again inside the desktop platform.

**Why the desktop version matters:** Unlike browser-based tools, the desktop client gives you terminal access for running local commands, Claude Code, and other AI tools directly within the project.

---

## Step 2: Generate the Front-End Structure First

1. Before building any functionality, create the visual layout with placeholder data.
2. Prompt Orchids: "Hey Orchid, please create a simple mobile optimized web app for [your app concept]. Do not build the actual functionality yet. Just create placeholder data and sections so we have a better idea of how the app will look as we develop it further."
3. Wait for Orchids to generate the application.
4. Review the layout to understand the navigation flow and section structure.

**Why this matters:** Building structure first prevents the common mistake of jumping into features without a clear visual framework, which leads to messy, unorganized projects.

---

## Step 3: Build Core Features Iteratively

Add real functionality one feature at a time by prompting Orchids:

1. **Core feature:** "Hey, great job. Now please build the [main feature]. Allow users to [specific actions] and make the user experience as great as possible."
2. Wait for generation, then test the feature by entering data.
3. **Second feature:** "Build the [next feature], allow users to [actions], and create automated insights based on [previously logged data]."
4. Continue this pattern, building each feature on top of previous data.

**Example sequence from the video:**
- Period tracker with symptom logging
- Ovulation tracker with automated predictions based on period data
- Pregnancy tracker with week-by-week insights
- Cycles prediction widget on insights page
- Symptom checker for health conditions
- Personalized dashboard widgets summarizing all features

---

## Step 4: Integrate Claude Code via the Terminal Panel

1. Inside the Orchids desktop app, open a terminal panel.
2. Type `claude` to launch Claude Code (must be installed on your machine).
3. Once initialized, prompt Claude Code directly within your project: "Assess the application, then create a dark theme and button for the application."
4. Claude Code will review the entire app and implement changes.

**Key advantage:** This is unique to Orchids -- no other AI builder lets you bring your own AI subscriptions (Claude, ChatGPT, Gemini, Copilot) into the platform at actual API cost instead of marked-up rates.

---

## Step 5: Sync Desktop Project to Web

1. In the Orchids desktop app, click the three dots in the upper right corner.
2. Select "Sync" from the project options menu.
3. Wait for the upload to complete.
4. Open your browser and access the project through the Orchids web platform.
5. All features, layouts, and updates built locally are now available online.
6. Continue development in the browser from anywhere without losing momentum.

---

## Step 6: Build Advanced Features on Web

1. Continue prompting from the browser version.
2. Example: "Please build the insights page. Remove all placeholder data and build a comprehensive page showing insights about the user cycles. Call this widget the cycles prediction tracker."
3. Expand with additional features: "Expand the insight section by creating a symptom checker. Users should be able to check their symptoms for potential conditions like PCOS, endometriosis, fibroids, and perimenopause."
4. Tie everything together: "Finalize the app by creating widgets in the dashboard showing personalized insights about the user for each functionality we have made."

---

## Step 7: Final QA and Responsive Design Pass

1. Prompt Orchids: "Great job. Now, lastly, check the app for any potential issues and create the proper web version that resorts to our current app's design when it's in mobile."
2. The app will adjust for web view: layout scales naturally, sections spaced correctly, mobile-first design preserved.
3. Walk through the entire app as a user would: log entries, check dashboard updates, verify insights, test all features end to end.
